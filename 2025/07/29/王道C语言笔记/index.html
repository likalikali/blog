在使用 scanf 读取混合类型的数据（如整数、字符、浮点数）时，需要特别注意空格和换行符的处理方式。

格式说明：

5.3 数组越界

在变量窗口中输入sizeof(a)，可以看到数组a的大小为20字节，计算方法其实就是sizeof(int)*5：数组中有5个整型元素，每个元素的大小为4字节，所以共有20字节。访问元素的顺序是依次从a[0]到a[4]，a[5]=6、a[6]=7均为访问越界。下图显示了代码运行情况，从中看出，执行到第12行时，变量i的值被修改了，这就是访问越界的危险性——未对变量i赋值，其值却发生了改变！

数组另一个值得关注的地方是，编译器并不检查程序对数组下标的引用是否在数组的合法范围内。这种不加检查的行为有好处也有坏处，好处是不需要浪费时间对有些已知正确的数组下标进行检查，坏处是这样做将无法检测出无效的下标引用。一个良好的经验法则是：如果下标值是通过那些已知正确的值计算得来的, 那么就无须检查; 如果下标值是由用户输入的数据产生的, 那么在使用它们之前就必须进行检查, 以确保它们位于有效范围内。

全局变量存储在哪里？如图 1.1 所示，全局变量 i 存储在数据段，所以 main 函数和 print 函数都是可见的。全局变量不会因为某个函数执行结束而消失，在整个进程的执行过程中始终有效，因此工作中应尽量避免使用全局变量！在前几章中，我们在函数内定义的变量都称为局部变量，局部变量存储在自己的函数对应的栈空间内，函数执行结束后，函数内的局部变量所分配的空间将会得到释放。如果局部变量与全局变量重名，那么将采取就近原则，即实际获取和修改的值是局部变量的值。

这是因为 C语言中的函数参数传递是“值传递”（pass by value）。也就是说：

  • 当你调用 modify_num(a); 时，变量 a 的值（即 10）被复制了一份传给了 modify_num 函数的参数 b。

  • 所以 a 的值在 main 函数中不会被改变，仍然是 10。

🚨 问题所在

当你调用 modify_pointer(p, q) 时：

  • p 的值（即 NULL）被复制一份传递给 modify_pointer。
  • 在 modify_pointer 中，p 是这个复制品，和 main 中的 p 没有任何直接关系。
  • 所以在 modify_pointer 中写 p = q; 只是改变了局部变量 p 的值，不会影响 main 中的 p。

因此，main 中的 p 依然是 NULL，你执行 *p 会导致 段错误（Segmentation Fault）。（操作系统根据空指针找不到对应的地方抛出错误信号）

✅ 正确做法：传入指针的地址（即“指针的指针”）

在这个过程中：

  • *m 是主函数中的变量 p

子函数中的变量 实际含义
m 指向主函数中 p 的指针（即 int **m）
*m 就是主函数中的 p 本身（int *）
**m 就是主函数中 p 指向的值（int）

✅ 为什么这更直观、更不容易混淆？

如果你在子函数中也用变量名 p：

这很容易让人混淆主函数中的 p 和子函数中的 p。

比如说：

  • p 是 int ** —— 和 main 里的 p 类型不一样！
  • *p 是 main 的 p，但看起来像是当前函数的变量。

✅ 所以用 m 来表示子函数中的形参，确实能让人更清晰地看到指针的层级和意义。

🧠 类比一句通俗话（帮助记忆）

子函数拿到的是“指向主函数中 p 的地址”
所以：

  • *m 就是主函数的 p
  • **m 就是主函数的 *p

建议 原因
在子函数中不要用 p 命名 容易和主函数中的 p 混淆
用 m、ptr_to_p 等中间变量名 有助于理解 *m = q 是在修改主函数中的指针
*m = 主函数的 p，**m = 主函数的 *p 这就是理解二级指针的关键
